Quick Start

How to use the dashboard

User friendly Compact layout Protection aware
Step 1
Read the top banner first

The large banner is the fastest summary. When live packets stop updating for about 30 seconds, the page treats the device as stale or disconnected so old values are not mistaken as current.

Step 2
Check System Overview

Use this area for overall health, relay control, refresh, and clear-fault actions. It is the main place to confirm whether the device is healthy, idle, active, or offline.

Step 3
Review the live cards

Voltage, current, apparent power, temperature, and waveform features help you understand both electrical load behavior and arc-related signal behavior.

Step 4
Treat trips seriously

If the device trips on heat, voltage, overload, or arc-related behavior, remove the cause first. Do not use the dashboard as a way to ignore a real fault.

Dashboard Overview

What each section means

Status Banner

The top banner shows the effective operating state, such as monitoring, unplugged, disconnected, overvoltage, undervoltage, heating, overload, or arcing.

Use this first before reading the smaller cards.

System Overview

This combines the current health summary with the main control actions. It helps you confirm if the device is healthy, active, idle, or offline, and it gives fast access to refresh, relay, and fault-clear actions.

Status

This block summarizes connection state, protection state, firmware version, and the latest event. It is the best place to confirm whether the page is seeing fresh data from the device.

Live Electrical

This shows voltage, current, apparent power, and temperature from the latest live packet. The small hints under each value are quick interpretations, not full diagnostics.

Time and Frequency Features

The dashboard separates feature values into Time Domain and Frequency Domain so the user can review signal behavior more clearly.

History

The history table shows recorded status events with time, voltage, current, temperature, and duration. The current dashboard supports Today, Yesterday, Last 7 days, and Last 30 days, plus CSV download.

The current dashboard also includes admin-only maintenance sections: OTA Updates, Data Logger, and Plot Viewer.
Protection Rules

What can alarm or trip the device

Overload alarm10 A and above
Sustained overload trip14.5 A for 60 s
Warm socket60 °C hint level
Heating trip70 °C and above
Normal mains207 V to 253 V

Arc Fault

Arc-related detection is based on waveform features and model logic rather than on one simple threshold alone. Treat this as a serious electrical condition.

Overload

The dashboard warns at high current. The earlier guide logic also documents a sustained-overload trip condition after extended heavy current.

Socket Heating

Temperature is tracked continuously. Warm values deserve attention, and higher values trigger protective action.

Undervoltage

Undervoltage now uses staged timing. Around 200 V to 207 V it must persist much longer, around 170 V to 200 V it trips after several seconds, and below about 170 V it is treated as a fast trip. Values collapsing toward 0 V are treated as unplugged or mains-loss behavior, not as undervoltage.

Overvoltage

High mains voltage now uses staged timing too. Around 250 V to 265 V it must persist for several seconds, while above about 265 V it is treated as a fast trip.

Unplugged or no mains

Very low measured voltage is treated separately from normal voltage faults so the UI can distinguish absent mains from unhealthy mains. A 0 V style collapse is not shown as undervoltage.

Waveform Feature Guide

What the feature cards mean

TIME

Cycle NMSE

Shows how different one cycle is from the next. Higher inconsistency can suggest unstable conduction.

TIME

ZC Variance

Measures how irregular the zero-cross timing is. Arcs can disturb normal behavior near crossing points.

TIME

ZC Dwell Ratio

Shows how long the signal stays near the zero region. Abnormal dwell can be useful during arc review.

TIME

Pulse Count / Cycle

Counts pulse-like activity inside each cycle. Unstable loads can create more pulse-like behavior.

TIME

Peak Fluctuation CV

Describes how much cycle peaks vary. Large variation usually means the waveform is less repeatable.

FREQ

HF Energy Ratio

Compares high-frequency energy to the rest of the signal. Arcing often increases high-frequency content.

FREQ

Wavelet Entropy

Shows how spread or disordered the signal becomes across wavelet bands.

FREQ

Spectral Entropy

Measures how random or disorderly the frequency spectrum looks.

FREQ

Midband RMS

Highlights residual mid-frequency content after smoother expected behavior is removed.

FREQ

THD Current

Total harmonic distortion is a supporting distortion metric. It should not be treated as a complete arc decision by itself.

Controls and Admin Tools

What the buttons and extra tools do

Relay and fault controls Everyday controls

The relay uses pulse-based control. The page sends short ON and OFF requests instead of continuously holding the relay state.

  • Relay ON sends a pulse request to close or re-enable the load path.
  • Relay OFF sends a pulse request to open or disable the load path.
  • Clear Fault sends a fault-clear request. This does not mean the electrical cause is already safe.
  • During critical or offline states, the page disables relay control buttons.
OTA Updates Admin mode only

OTA lets the admin publish a target firmware version and firmware URL so the device can pull the update.

  • View the currently published OTA version and link.
  • Set a version string and firmware bin name.
  • Apply a repo-based raw firmware link, then publish the update.
Data Logger Admin mode only

The Data Logger is used for waveform collection and model-training workflows.

  • Set collection duration in seconds.
  • Set load type text for the session.
  • Choose a label override or leave it in auto mode.
  • Download the current session or compile all sessions.
  • Upload an external CSV into the Plot Viewer.
  • Clear all ML logs and sessions when needed.
Plot Viewer Admin mode only

The Plot Viewer is used to inspect logged sessions visually.

  • Play or pause time playback.
  • Change speed and scrub through the session.
  • Choose which series to show.
  • Enable smoothing, normalization, statistics, event markers, and zoom reset.
What To Do After a Fault

Safe response guide

Arc FaultStop using the appliance

Unplug the load and inspect the plug, cord, socket, and appliance for burning smell, carbonization, looseness, or intermittent contact.

  • Avoid repeated reconnection tests.
  • Replace suspicious cords, plugs, or outlets before reuse.
Overload or sustained overloadReduce the load

Separate high-draw appliances and verify that current returns to a safer level before continuing use.

  • Avoid stacking heaters, irons, kettles, or similar heavy loads on the same outlet path.
  • Avoid undersized extension cords.
Socket heatingCheck the outlet physically

Allow the outlet and plug to cool. Inspect for loose grip, discoloration, melted plastic, corrosion, or blade wear.

  • Do not keep using a warm outlet just because the appliance still powers on.
  • Replace worn sockets and plugs.
Undervoltage or overvoltageWait for mains to stabilize

Low or high mains can stress motors, compressors, adapters, and appliance electronics.

  • Avoid repeated restarts during unstable supply conditions.
  • For recurring overvoltage, inspect the power source or use proper surge protection.
General rule: never treat a protection trip as something to bypass. Remove the cause first, then reset only after you have confirmed the appliance and outlet are safe.
Reset Guidance

How to reset after a trip

Before resetting

  1. Read the fault shown on the dashboard.
  2. Inspect the appliance, plug, socket, and nearby area.
  3. Remove or correct the suspected cause first.

After resetting

  1. Reconnect only when you believe the condition is safe.
  2. Watch the banner and overview again.
  3. If the same fault returns, stop using the appliance and inspect more thoroughly.
The current dashboard logic disables remote relay controls during critical states, and the existing guide logic documents that voltage faults may auto-reclose only after stable mains has returned for several minutes.
